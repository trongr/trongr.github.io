Genetic Programming: Overview

A genetic algorithm

  1. Randomly generate a collection of chromosomes, called the population.
  2. Evaluate each chromosome in the population to determine fitness.
  3. Based on the fitness, select members of the population that will be used to create the next generation of the population, i.e. selecting who will get to breed children. One strategy for creating children is called recombination, or crossover, where a child is created by combining the genes of the two parents.
  4. Mutate some parents and offspring.
  5. Form new population from mutated parents and offspring.
  6. Optional: promote some unaltered parents from to the next generation. This process is called Elitism.
  7. Repeat as required (until max number of generation or fitness is acceptable).

Bloat

As a population evolves over time, the chromosomes may grow in size without improving fitness. This is similar to how real software becomes bloated over time as you add more and more code to a project. Productivity stagnates because the system has to overcome friction created by old code, e.g. it's hard to remove old legacy code, data has to flow through more and more code in order to get from input to output, etc.
